Semantic UI

Semantic UI är ett CSS-ramverk som gör det möjligt att skapa responsiva webbsidor med hjälp av "mänskligt läsbar" HTML. Ramverket använder semantiska klassnamn, vilket gör koden mer intuitiv och lättare att förstå för utvecklare. Det erbjuder ett stort antal inbyggda komponenter, inklusive knappar, formulär och navigeringsmenyer, som gör det enkelt att bygga ett användargränssnitt snabbt.

Fördelar

Jag tycker att Semantic UI var relativt lätt att komma igång med då ramverket har en bra dokumentation. Det var lätt att följa dokumentationen och fanns bra exempel för att förstå hur "slutresultatet" av en viss komponent skulle se ut och bete sig. Något annat som jag gillar med Semantic UI är namnen på elementen. Eftersom namnen faktiskt speglar vad elementet är blir det enklare att förstå, som exempelvis UI header, UI table, UI menu med mera.

Nackdelar

De nackdelar som jag upptäckt hittils är dels att det inte går att söka i dokumentationen. Detta är något som jag saknat då det blir svårare att hitta när man endast kan navigera i menyn. Jag upplever även att den margin/padding som är standard för Semantic UI:s element är väldigt liten, vilket har krävts egna justeringar för att fixa detta. Det ser jag som negativt då det kräver extra tid för utveckling och anpassning.

Jag tycker inte heller att responsiviteten är den bästa. Det kan dock bero på min okunnighet, men vad jag kunde hitta fanns bara ett alternativ för mobilmeny vilket var "stackable". Detta är inte optimalt om man har väldigt många navigeringslänkar i menyn. Dessutom fungerade det inte för mig att ha en sökfunktion i menyn, då sidan inte blev responsiv.
